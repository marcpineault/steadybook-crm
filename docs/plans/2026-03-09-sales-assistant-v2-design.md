# Calm Money Sales Assistant v2 — Design

## Overview

Expand the existing Telegram pipeline bot into a proactive sales assistant that helps Marc close more deals by automating follow-ups, drafting emails, processing meeting notes, and managing prospecting calls.

## Current System

- Telegram bot on Railway + live web dashboard
- Pipeline CRM in Excel (add/update/log/export/upload)
- Claude Haiku processes natural language commands
- Products: Life Insurance, Wealth Management

## New Features

### 1. Morning Briefing (Scheduled — 8 AM ET)

Bot texts Marc every morning with:

- Follow-ups due today (name, what's needed, priority)
- Overdue follow-ups (name, days late)
- Meetings today with prep notes (pulled from pipeline notes + transcripts)
- Insurance book calls queued for the day
- Pipeline snapshot (active deals, value, hot leads)

If nothing is due, send a shorter "all clear" message.

Marc can reply directly to take action ("done with Sarah", "push Michael to Friday").

**Implementation:** Use APScheduler with CronTrigger. Runs `get_morning_briefing()` at 8 AM ET. Sends via Telegram bot API.

### 2. Email Drafting

Commands:
- `draft follow-up for [name]` — uses pipeline notes + stage context
- `draft quote email for [name], [prices]` — formats quote in Marc's casual style
- `draft intro email for [name], referral from [source]`
- `draft check-in for [name]`

Style: casual, direct, short. Sign off as "Marc / Calm Money".

Bot replies with the full email. Marc copy-pastes into Outlook.

**Implementation:** Add `draft_email` tool. System prompt includes Marc's email style guide and examples. Claude generates based on prospect context from pipeline.

### 3. Otter Transcript Processing

Flow: Otter records meeting → Zapier auto-sends transcript to Telegram bot.

Bot receives transcript and:
1. Summarizes key points (financial situation, needs, concerns)
2. Saves summary to prospect's notes in pipeline (appends, doesn't overwrite)
3. Drafts follow-up email based on discussion
4. Identifies and sets next steps with dates
5. Logs the meeting in Activity Log

Marc can also paste transcripts manually as fallback.

**Implementation:** Detect long messages (>500 chars) or messages starting with transcript-like patterns. Process with Claude to extract structured data. Zapier webhook sends to a dedicated endpoint or directly to the Telegram bot via API.

### 4. Insurance Book Prospecting System

Setup: Marc uploads home insurance book as CSV/Excel to Telegram.

New Excel sheet: "Insurance Book" with columns:
- Name, Phone, Address, Policy Start, Status (Not Called, No Answer, Not Interested, Booked Meeting, Callback, Client), Last Called, Notes, Retry Date

Daily flow:
- Morning briefing includes 5 calls from the book (configurable)
- Serves prospects in order, skipping completed ones
- After each call, Marc texts outcome:
  - `called [name], not interested` → marked done
  - `called [name], booked meeting [date]` → auto-adds to pipeline as New Lead
  - `called [name], no answer` → retry in 3 days
  - `called [name], callback next week` → retry date set
  - `skip today` → pauses queue

Stats command: `book stats`
- Total in book, called, remaining
- Meetings booked, conversion rate
- Calls this week/month

**Implementation:** Add new sheet to pipeline.xlsx. New tools: `get_next_calls`, `log_book_call`, `get_book_stats`, `upload_book`. Track position with a simple counter.

### 5. Auto-Nag System (Scheduled checks)

Runs every 2 hours during business hours (9 AM - 5 PM ET):

| Trigger | Message |
|---------|---------|
| No activity on prospect for 7+ days | "Haven't touched [name] in X days. Draft a check-in?" |
| Follow-up 2+ days overdue | "[name] follow-up is X days late. Call or push?" |
| Hot lead in New Lead/Contacted for 5+ days | "[name] is Hot but still early stage. Book a meeting?" |
| Meeting tomorrow | "Reminder: [type] with [name] tomorrow at [time]. Notes: [summary]" |

Marc can reply: "draft email", "push to friday", "done", "snooze" (reminds tomorrow).

Nag tracking: don't repeat the same nag within 24 hours.

**Implementation:** APScheduler interval trigger every 2 hours. Track last nag time per prospect to avoid spam. Store nag state in a simple JSON file.

### 6. Meeting Tracker

Commands:
- `meeting with [name] [day] [time] [type]`
- `what's on this week?`
- `cancel meeting with [name]`

Storage: New "Meetings" sheet in pipeline.xlsx
- Columns: Date, Time, Prospect, Type (Discovery, Plan Presentation, Review, Other), Prep Notes (auto-pulled), Status (Scheduled, Completed, Cancelled)

Included in morning briefing with auto-generated prep notes from pipeline data.

**Implementation:** New tools: `add_meeting`, `get_meetings`, `cancel_meeting`. Prep notes generated by reading prospect's pipeline entry + any saved transcript summaries.

## Architecture

```
Marc's Phone (Telegram)
    ↓
Railway (Python)
    ├── bot.py — Telegram handler + Claude tool calls
    ├── dashboard.py — Flask web dashboard
    ├── scheduler.py — APScheduler (morning briefing, auto-nags)
    └── pipeline.xlsx
        ├── Pipeline (existing)
        ├── Dashboard (existing)
        ├── Activity Log (existing)
        ├── Weekly Review (existing)
        ├── Insurance Book (new)
        └── Meetings (new)

Zapier: Otter → Telegram Bot API
```

## New Dependencies

- `apscheduler` — scheduled tasks (morning briefing, nags)
- `pytz` — timezone handling (ET)
