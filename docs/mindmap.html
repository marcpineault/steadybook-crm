Data Sources — What Feeds In
Every way information enters the system
Telegram Primary UI
Your main interface. Commands, voice notes, documents, free-form messages. Everything you type or speak goes through here.
You → Telegram → bot.py → SQLite
Otter.ai Via Zapier
Records and transcribes your calls. Zapier watches for new Otter transcripts, sends them to your Telegram chat. The bot auto-detects "this is an Otter transcript" and processes it.
Phone call → Otter records → Zapier → Telegram msg → bot detects Otter → extract & log
Outlook Bookings Power Automate
When a prospect books a meeting through your Outlook booking page, Power Automate fires a webhook with their name, email, phone, service, and time.
Prospect books → Outlook → Power Automate → /api/intake → new prospect + meeting
Outlook Calendar Power Automate
Calendar events with attendees trigger a webhook. Each attendee is added as a prospect. Meeting type auto-classified from the subject line.
Calendar event → Power Automate → /api/intake → prospects + meeting + prep doc
CloudMailin Email Forwarding
Inbound emails forwarded to a CloudMailin address. Parsed and sent to your bot's /api/email-inbound webhook. GPT extracts the lead from the email body.
Email received → forward to CloudMailin → /api/email-inbound → GPT extraction → new prospect
Zapier Email Leads Zapier
Zapier catches specific emails (referrals, web form submissions) and POSTs them to /api/intake as an email_lead type. GPT extracts prospect data.
Gmail/Outlook trigger → Zapier → /api/intake (email_lead) → GPT extract → prospect
Voice Notes Whisper
Send a voice note in Telegram after a call or meeting. Whisper transcribes it, GPT extracts prospects, facts, and action items. Follow-up auto-drafted.
Voice .ogg → Whisper API → transcript → GPT extract → prospect + memory + follow-up draft
Web Dashboard Streamlit/Flask
Browser-based dashboard for visual pipeline management. Can add prospects, log activities, manage tasks via the web UI and REST API.
Browser → dashboard.py (Flask) → /api/* endpoints → SQLite

AI Processing — The Brains
OpenAI models that power every intelligent feature
GPT-4.1 Strategic
Heavy lifting: follow-up drafts, content generation, briefings, campaign messages, nurture touches, meeting prep, analytics insights. Warm, Marc-sounding output.
GPT-4.1 Mini Fast
Lightweight tasks: compliance checking, email extraction, lead data parsing, audience segmentation, task creation from natural language. Speed over depth.
GPT-5 Chat
Powers the free-form chat in Telegram. When you ask a question or have a conversation that isn't a command, this model responds.
Whisper Transcription
Converts voice notes (.ogg files from Telegram) into text. Feeds into the extraction pipeline for prospect creation and memory building.

Core System — The 5 Phases
15 Python modules working together
Phase 1: The Memory
memory_engine.py + voice_handler.py — Extracts and stores facts about every client across 6 categories. Powers all personalization downstream.
Phase 2: The Communicator
follow_up.py + compliance.py + approval_queue.py + briefing.py — Auto-drafts, compliance-checks, and queues every outgoing message for your approval.
Phase 3: The Marketer
content_engine.py + market_intel.py + meeting_prep.py — Social content in your voice, market calendar awareness, pre-meeting prep docs.
Phase 4: The Outreach Rep
campaigns.py + nurture.py + intake.py — Batch campaigns, 4-touch nurture sequences, multi-source lead intake with auto-scoring.
Phase 5: The Analyst
analytics.py + scoring.py + dashboard.py — Outcome tracking, response rates, conversion rates, AI insights that feed back into strategy.
Database: SQLite + WAL
15 tables: prospects, activities, meetings, tasks, client_memory, approval_queue, audit_log, outcomes, campaigns, nurture_sequences, brand_voice, market_calendar, trust_config, insurance_book, win_loss_log.

Hosting & Infrastructure
Where it all runs
Railway Hosting
Docker container running Python 3.13. Flask serves the dashboard + webhook endpoints. Telegram bot runs in the background with APScheduler. Persistent volume for SQLite.
SQLite + WAL Database
15 tables, file-based, concurrent-safe with Write-Ahead Logging. Lives on Railway persistent volume. No external database needed.
Webhook Mode Production
Telegram uses webhook (not polling) in production. Railway domain registered as webhook URL. Flask route /webhook handles incoming updates.

Security & Compliance
How everything stays safe and auditable
Webhook Auth HMAC
Every webhook endpoint requires a secret header (INTAKE_WEBHOOK_SECRET, CLOUDMAILIN_SECRET). Rejects unauthorized requests.
AI Compliance Gate Every Message
GPT-4.1 Mini scans every client-facing message for: false return promises, misleading claims, missing disclaimers, tone violations. Logged to audit_log.
Human Approval Required
No AI-generated message reaches a client without Marc tapping Approve in Telegram. Full audit trail of every draft, approval, and dismissal.
Dashboard Security CSRF + CSP
CSRF tokens (1-hour TTL), Content Security Policy headers, API key auth for programmatic access, 1MB payload limits.
